---
title: "Phone Number"
description: "Add SMS OTP and phone-number sign-in flows."
---

The phone-number plugin stores phone details on the user model, sends OTP codes through an application-provided SMS sender, verifies codes, and supports phone-based password reset.

## Installation

~~~bash
uv add kernia
~~~

## Import path

~~~python
from kernia.plugins.phone_number import phone_number
~~~

## Server configuration

~~~python title="auth.py"
import os

from kernia import KerniaOptions
from kernia.auth import init
from kernia.plugins.phone_number import phone_number

from .db import adapter

auth = init(KerniaOptions(
    database=adapter,
    secret=os.environ["KERNIA_SECRET"],
    base_url=os.environ["KERNIA_BASE_URL"],
    base_path="/api/auth",
    plugins=(
        phone_number(),
    ),
))
~~~

## API routes

<APIMethod method="POST" path="/api/auth/sign-in/phone-number">
  Signs in with phone number and password.
</APIMethod>

<APIMethod method="POST" path="/api/auth/phone-number/send-otp">
  Sends an OTP to a phone number.
</APIMethod>

<APIMethod method="POST" path="/api/auth/phone-number/verify">
  Verifies a phone OTP.
</APIMethod>

<APIMethod method="POST" path="/api/auth/phone-number/request-password-reset">
  Sends a reset OTP for a phone-number account.
</APIMethod>

<APIMethod method="POST" path="/api/auth/phone-number/reset-password">
  Consumes the reset OTP and updates the password.
</APIMethod>

## Schema impact

Extends the user schema with phone-number fields through `phone_number_schema` and reuses verification data for OTPs.

## Behavior and options

- Configure `advanced["phone-number"]` with `send_sms`, `otp_length`, `expires_in`, and `disable_sign_up`.
- Normalize phone numbers before storage and display.
- Keep send and verify routes rate-limited.

## Test and demo coverage

Test send, verify, sign-in, reset, invalid OTP, expiry, missing sender, normalized phone matching, and frontend unavailable states when SMS is not configured.
