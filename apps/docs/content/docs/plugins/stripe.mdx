title: "Stripe"

---

The Stripe plugin connects Kernia users or organizations to Stripe customers and subscriptions. It keeps the compatible checkout, portal, cancel, resume, list, and webhook surfaces, and adds a Kernia billing layer for imported products and prices, feature entitlements, usage events, `check`, and `track`.

## Installation

```bash
uv add kernia-stripe
```

## Server configuration

```python title="auth.py"
import os

from kernia import KerniaOptions
from kernia.auth import init
from kernia_stripe import StripeClient, StripeOptions, StripePlan, stripe

from .db import adapter

stripe_client = StripeClient(api_key=os.environ["STRIPE_SECRET_KEY"])

auth = init(KerniaOptions(
    database=adapter,
    secret=os.environ["KERNIA_SECRET"],
    base_url=os.environ["KERNIA_BASE_URL"],
    plugins=(
        stripe(StripeOptions(
            stripe_client=stripe_client,
            webhook_secret=os.environ["STRIPE_WEBHOOK_SECRET"],
            subscription_for="user",
            plans={
                "pro": StripePlan(
                    name="pro",
                    price_id="price_monthly_pro",
                    annual_price_id="price_annual_pro",
                ),
                "team": StripePlan(
                    name="team",
                    price_id="price_team_base",
                    seats=True,
                    seat_price_id="price_team_seat",
                ),
            },
        )),
    ),
))
```

<Callout type="info" title="Import-first catalog">
  Kernia imports Stripe products and prices into local billing tables first. You can then map imported prices to internal plans, features, entitlements, and UI copy without pretending Stripe is the only billing source of truth.
</Callout>

## Checkout and subscription routes

<APIMethod method="POST" path="/api/auth/stripe/checkout-session">
  Requires a session. Body: `{ "plan": "pro", "successUrl": "https://app.example.com/billing/success", "cancelUrl": "https://app.example.com/billing", "referenceId": "org_123", "seats": 5 }`. Returns `{ "url", "id" }`.
</APIMethod>

<APIMethod method="POST" path="/api/auth/stripe/billing-portal">
  Requires a session. Body: `{ "returnUrl": "https://app.example.com/billing", "referenceId": "org_123" }`. Returns `{ "url" }`.
</APIMethod>

<APIMethod method="POST" path="/api/auth/stripe/cancel-subscription">
  Requires a session. Body: `{ "subscriptionId": "sub_123", "cancelAtPeriodEnd": true }`.
</APIMethod>

<APIMethod method="POST" path="/api/auth/stripe/resume-subscription">
  Requires a session. Body: `{ "subscriptionId": "sub_123" }`.
</APIMethod>

<APIMethod method="GET" path="/api/auth/stripe/list-subscriptions">
  Requires a session. Optional query: `referenceId=org_123`. Returns local subscription rows.
</APIMethod>

<APIMethod method="POST" path="/api/auth/stripe/webhook">
  Verifies the `Stripe-Signature` header, persists subscription lifecycle changes, updates status on invoice events, and calls `on_event` when configured.
</APIMethod>

## Catalog sync

<APIMethod method="POST" path="/api/auth/stripe/catalog/sync">
  Requires a session. Imports Stripe products into `billingProduct`, prices into `billingPrice`, and writes a `billingSyncState` row for the `stripe` source.
</APIMethod>

<APIMethod method="GET" path="/api/auth/stripe/products">
  Requires a session. Returns `{ "products": [...] }` from the local `billingProduct` table.
</APIMethod>

<APIMethod method="GET" path="/api/auth/stripe/prices">
  Requires a session. Returns `{ "prices": [...] }` from the local `billingPrice` table.
</APIMethod>

## Billing and entitlements

<APIMethod method="POST" path="/api/auth/billing/check">
  Requires a session. Body: `{ "feature": "projects", "referenceId": "org_123", "required": 1 }`. Returns allowance, included quantity, used quantity, remaining balance, and overage state.
</APIMethod>

<APIMethod method="POST" path="/api/auth/billing/track">
  Requires a session. Body: `{ "feature": "projects", "referenceId": "org_123", "quantity": 1, "properties": { "project_id": "prj_123" } }`. Creates a usage event and updates entitlement usage.
</APIMethod>

<APIMethod method="GET" path="/api/auth/billing/customer">
  Requires a session. Ensures a Stripe customer exists for the active user and mirrors it into `billingCustomer`.
</APIMethod>

<APIMethod method="GET" path="/api/auth/billing/portal">
  Requires a session. Query: `returnUrl=https://app.example.com/billing`. Returns a Stripe Billing Portal URL.
</APIMethod>

<APIMethod method="GET" path="/api/auth/billing/usage">
  Requires a session. Optional query: `referenceId=org_123`. Returns usage events for the billing reference.
</APIMethod>

## Feature types

| Type | Model behavior |
|---|---|
| Boolean | Represented by an entitlement with `unlimited=True` or an included grant. |
| Metered consumable | `billingUsageEvent` rows increase `billingEntitlement.used`. |
| Quantity | `included` grants a non-consumable allowance. |
| Seat-style | `StripePlan(seats=True)` lets organization seat sync update Stripe quantity when wired with the organization plugin. |
| Overage | `overageAllowed=True` lets `billing/check` allow usage beyond included balance. |

## Schema changes

The plugin contributes these tables: `subscription`, `billingProduct`, `billingPrice`, `billingFeature`, `billingPlan`, `billingPlanFeature`, `billingEntitlement`, `billingUsageEvent`, `billingCustomer`, and `billingSyncState`. It also extends `user` with `stripeCustomerId`.

## Webhooks

Configure Stripe to send events to:

```text
https://api.example.com/api/auth/stripe/webhook
```

At minimum, enable:

- `checkout.session.completed`
- `customer.subscription.created`
- `customer.subscription.updated`
- `customer.subscription.deleted`
- `invoice.paid`
- `invoice.payment_failed`

## Current limits

The billing layer has import, check, track, portal, customer, and usage primitives. Plan-feature mapping and entitlement grants should be wired by your admin UI or backend job before a production SaaS claims Autumn-style billing parity end to end.

## Troubleshooting

- `PLAN_NOT_FOUND`: the checkout `plan` is not present in `StripeOptions.plans`.
- `REFERENCE_REQUIRED`: `subscription_for="organization"` requires `referenceId`.
- `INVALID_SIGNATURE`: the webhook secret does not match the Stripe endpoint secret or the payload was modified before verification.
- Usage is tracked but still denied: create or update a `billingEntitlement` row for that `referenceId` and `featureKey` before calling `track`.
