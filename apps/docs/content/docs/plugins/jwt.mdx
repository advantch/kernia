---
title: "JWT"
description: "Issue local JWT access tokens and publish JWKS keys."
---

The JWT plugin creates and rotates signing keys stored in the database, issues short-lived access tokens for active sessions, and publishes public keys through a JWKS route. Other plugins, including MCP, can use the same signer.

## Installation

~~~bash
uv add kernia authlib
~~~

## Import path

~~~python
from kernia.plugins.jwt import JwtOptions, jwt
~~~

## Server configuration

~~~python title="auth.py"
import os

from kernia import KerniaOptions
from kernia.auth import init
from kernia.plugins.jwt import JwtOptions, jwt

from .db import adapter

auth = init(KerniaOptions(
    database=adapter,
    secret=os.environ["KERNIA_SECRET"],
    base_url=os.environ["KERNIA_BASE_URL"],
    base_path="/api/auth",
    plugins=(
        jwt(JwtOptions(issuer=os.environ["KERNIA_BASE_URL"])),
    ),
))
~~~

## API routes

<APIMethod method="GET" path="/api/auth/token">
  Issues a JWT for the active session.
</APIMethod>

<APIMethod method="GET" path="/api/auth/jwks">
  Publishes public JSON Web Keys for token verification.
</APIMethod>

<APIMethod method="POST" path="/api/auth/jwks/rotate">
  Marks current keys inactive and creates a new active signing key.
</APIMethod>

## Schema impact

Adds the `jwk` table with key id, algorithm, public key, private key, active flag, and timestamps.

## Behavior and options

- Default access token lifetime is 15 minutes.
- Supported algorithms include ES256, RS256, and EdDSA depending on installed crypto support.
- Protect rotation with an admin token or an authenticated admin session.

## Test and demo coverage

Test first-use key creation, token issuance, JWKS output, key rotation, old-key verification, unauthorized rotation, and issuer/audience claims.
