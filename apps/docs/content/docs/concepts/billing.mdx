`kernia-stripe` keeps Better Auth-compatible subscription routes and adds a
Kernia billing layer for SaaS products.

The model is Stripe-import-first:

```http
POST /api/auth/stripe/catalog/sync
```

Kernia imports Stripe products and prices, then lets your application map those
records to plans, features, entitlements, and usage rules.

## Runtime checks

```http
POST /api/auth/billing/check
POST /api/auth/billing/track
GET  /api/auth/billing/usage
```

Supported feature types include boolean features, metered consumables,
non-consumable quantities, included grants, reset periods, overage flags, and
seat-style quantities.
